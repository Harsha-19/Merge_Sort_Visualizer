## Proposed Architecture & Tech Stack

### Frontend (React + Vite + TypeScript)
- **Styling**: Tailwind CSS + shadcn/ui components
- **Animation**: Framer Motion for smooth array bar transitions
- **State Management**: Zustand for global playback control
- **Icons**: Lucide React

### Backend (Node.js + Express)
- Minimal Express server to serve quiz questions and validate answers.
- Static JSON data for the quiz.

## Implementation Steps

### Phase 1: Project Initialization
1. Initialize the `frontend` directory with Vite (React + TS).
2. Configure Tailwind CSS and setup shadcn/ui.
3. Install required frontend libraries: `framer-motion`, `zustand`, `lucide-react`, `clsx`, `tailwind-merge`.
4. Initialize the `backend` directory with Node.js and Express.
5. Set up basic backend routes (e.g., `/api/quiz`).

### Phase 2: Core Algorithm Engine
1. Implement the top-down Merge Sort algorithm generator.
2. It will record snapshots of every step with the following structure:
   - `type`: "DIVIDE" | "CONQUER" | "MERGE"
   - `array`: Current state of the array
   - `activeIndices`: Indices currently being processed or compared
   - `depth`: Recursion depth
   - `explanation`: Contextual text for the step
   - `analogy`: Associated real-world analogy text

### Phase 3: State Management (Zustand)
1. Create a global store to manage:
   - Current array input (default: `[8, 3, 5, 2]`)
   - Array of all steps generated by the engine
   - Current step index
   - Playback state (isPlaying, speed)
2. Implement actions: `play`, `pause`, `stepForward`, `stepBackward`, `reset`, `setSpeed`, `setArray`.

### Phase 4: Frontend UI Components
1. **Layout**: Header (Title + Input System), Main Content, Footer (Complexity + Quiz).
2. **Visualizer Component**: Vertical bars using Framer Motion. Heights correspond to values. Colors change based on state (blue for default, yellow for comparing, green for merging).
3. **Analogy & Phase Panel**: Displays the current phase ("DIVIDE", "CONQUER", "MERGE") and real-time analogy.
4. **Controls System**: Playback buttons (Play, Pause, Step Forward/Backward, Reset, Speed Slider).
5. **Complexity Panel**: Static informational component explaining Time/Space complexity and stability.
6. **Quiz Module**: Component that fetches data from the backend API, tracks answers, and shows the score.

### Phase 5: Integration & Polish
1. Connect all components to the Zustand store.
2. Implement strict responsive layout and SaaS-quality aesthetics (soft shadows, rounded borders, clean spacing).
3. Ensure animations perform smoothly under 100ms per frame.
